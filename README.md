# Inefficiency Analysis and Optimization of Theoretical Production Times through Machine Learning

## Project Objective 🎯 

This project develops machine learning models to analyze and predict production inefficiencies in industrial machinery, improve the estimation of theoretical processing times, and calculate the Overall Equipment Effectiveness (OEE) index. Using regression and classification techniques, the system identifies operational anomalies, redefines theoretical times more consistently with actual times, and provides interactive dashboards for production monitoring. The goal is to support industrial decision-making with quantitative and replicable predictive tools.

## 1. Features 📊

### Regression
- Prediction of the **Inefficiency Index** using a regression model.
- Prediction of **Processing Time (HOURS)** with comparison to theoretical time (AS400).
- Prediction of the **OEE index** before a work order is executed, using a dedicated regression model.

### Classification
- Standard classification into 3 classes:
  - **NORMAL**
  - **WARNING**
  - **ANOMALY**
- Anomaly-oriented classification optimized to detect anomalous cases.
- Custom-threshold classification combining tuned thresholds for anomaly detection.

### OEE Analysis
- Calculation of the three OEE components:
  - **Availability** (A): scheduled time minus downtime.
  - **Performance** (P): theoretical cycle time versus actual cycle time.
  - **Quality** (Q): good pieces over total pieces produced.
- OEE class assignment: **Critico** (< 65%), **Accettabile** (65–85%), **Ottimo** (≥ 85%).
- Automatic alert generation for work orders where OEE, Performance, Availability, or Quality fall below configurable thresholds.

### Feature Engineering
- Dedicated pipeline for all models:
  - Temporal, Ratio, Lag and Rolling window features.
  - OEE-specific features: setup fraction, downtime fraction, per-article cycle statistics, rolling OEE, lagged OEE values.

### Dashboards
- **OEE Dashboard** (`outputs/dashboard_oee.html`): self-contained interactive HTML page with KPI cards (mean OEE, Availability, Performance, Quality), monthly trend charts, class distribution, worst work orders table, and per-article OEE ranking. Generated by `src/OEE/dashboard.py`.
- **Anomaly Dashboard** (`outputs/dashboard_anomaly.html`): self-contained interactive HTML page with anomaly KPIs, an interactive monthly calendar (click a day to see its work orders), anomaly rates by article and phase, a forecast table for next-month risk, and the full work-order list with probability-based status badges. Generated by `src/classification/dashboard.py`.

### Other
- Automatic saving of all predictions to CSV (`outputs/predizioni.csv` or a custom path).
- Separate scripts for model training/tuning and a single entry point (`main.py`) for inference.

## 2. Installation 📦

### Prerequisites

- Python 3.10+
- Pip updated
- Virtual Environment (venv) recommended

### - Clone the repository

```bash
    git clone <URL_REPOSITORY>
    cd tesi_efficienza_macchine
```

### - Create and activate the virtual environment

```bash
    python -m venv venv
```

#### Windows:

```bash
    .\venv\Scripts\Activate.ps1
```

#### Linux/macOS:

```bash
    source venv/bin/activate
```

### - Install dependencies

```bash
    pip install --upgrade pip
    pip install -r requirements.txt
```

## 3. Usage 🚀

#### - Run predictions (inference)

Runs all six models (inefficiency regression, time regression, two anomaly classifiers, custom-threshold classifier, OEE regression) and saves a combined CSV.

- Use the default dataset

```bash
    python main.py
```

- Use a custom CSV

```bash
    python main.py --input data\processed\koepfer_160_2.csv
```

- CSV input + custom output path

```bash
    python main.py --input data\processed\koepfer_160_2.csv --output outputs\predizioni_custom.csv
```

#### - Generate dashboards

- OEE dashboard (historical KPIs, trends, worst work orders):

```bash
    python src\OEE\dashboard.py
```

- Anomaly detection dashboard (calendar, forecasts, article/phase breakdown):

```bash
    python src\classification\dashboard.py
```

Both commands produce standalone HTML files in `outputs/` that can be opened directly in any browser.

#### - Model training (optional)

```bash
    python src\regression\regression_inefficiency_models.py
    python src\regression\regression_time_models.py
    python src\classification\classification_models_standard.py
    python src\classification\classification_models_anomaly_oriented.py
    python src\classification\classification_models_anomaly_bigdata.py
    python src\classification\classificazione_soglie_custom.py
    python src\OEE\OEE_regression.py
```

## 4. Output Predictions CSV

The `predizioni.csv` file produced by `main.py` contains the original work-order identifiers alongside six prediction columns:

| Column | Description |
|---|---|
| `Indice_Inefficienza_Predetto` | Predicted inefficiency index (regression) |
| `Tempo_Predetto_ORE` | Predicted processing time in hours (regression) |
| `Classe_Anomaly_Oriented` | Binary anomaly classification (NORMALE / ANOMALIA) |
| `Classe_Anomaly_Oriented_Big_Data` | 3-class anomaly classification (NORMALE / ATTENZIONE / ANOMALIA) |
| `Classe_Soglie_Custom` | Custom-threshold binary classification (NORMALE / ANOMALIA) |
| `OEE_Predetto` | Predicted OEE index (regression, range 0–1) |
